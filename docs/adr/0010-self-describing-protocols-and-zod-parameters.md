# Protocol packages are self-describing and compose through injected dependencies

A Protocol package exports top-level self-describing `@Protocol` classes; the composition root supplies selected modules, and Registry registers their Protocol exports and recursively injects declared dependencies. Core owns framework contracts, simulator owns trace mechanics, Protocol packages own ABI semantics and Receipts, `@themoss/erc` stays address-free, system owns verified Monad constants and rejects non-143 RPCs, and MCP server owns transport; dynamic addresses come from chain state and adding a Protocol changes only its package and composition root. Capability and Query inputs use `{ type, description }`, separating a reusable context-free Zod value contract from the field's purpose and exposing the generated JSON-safe schema through `load`.

Protocol classes compose through declared `protocols` dependencies and typed injected fields. A decorated Protocol cannot extend another decorated Protocol; Registry rejects that inheritance so Receipt-only instances never execute a parent Protocol's Runtime-injecting constructor.
