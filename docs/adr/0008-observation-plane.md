# Simulation evidence has two planes: a closed audit and an authored observation

Simulation output is split (`PlanSimResult`):

- **Audit plane** (`effects` + `warnings`) — protocol-agnostic by construction: the generic event vocabulary (Transfer/Approval families, selectors DERIVED from signatures, canonical hashes pinned in tests) plus reconciliation against `expects`. CLOSED to protocol code: letting the protocol under test define how it is audited would kill the independence that makes declared-vs-actual meaningful.
- **Observation plane** (`observations`) — protocol-AUTHORED narration via the `@Event` decorator: a multi-event subscription (contract-key → event names, both autocompleted from the injected Handles' origin-verified ABIs), an optional `dealer` preprocessor (method name or function; filters/aggregates the matched batch), a handler shaping raw args into human values, and a result-intent template rendered from the handler's return (strict: missing placeholder throws). One auto-injected `ObserveCtx` ({plan, account, token, shared}) is shared per plan × protocol between dealer and handlers. The protocol class type argument is MANDATORY (`@Event<Kuru>`) — TS cannot infer a factory's type parameters from the decoration site, and the explicit argument is what powers completion.

**The red line (one-directional): observations may only TIGHTEN the outcome, never satisfy it.** The single crossing point is `confirms`: a capability may declare which observations are its on-chain receipts; a declared receipt that fails to appear (or an unwired observer) raises `CONFIRMATION_MISSING`. `confirms` is covered by `planHash` so it cannot be stripped in transit.

Protocol-specific receipts belong in the observation plane and must be wired through `@Event` + `confirms`. Canonical standard events that the closed audit plane already parses and fully binds to quantified `expects` — currently ERC-20/721/1155 Transfer families — are the narrow exception: duplicating them as authored observations would add no independent evidence. This exception does not extend to protocol-specific events or to standard events that are not reconciled against the Plan.

Why authored rather than auto-decoded from ABIs: raw event args are routinely in protocol-internal units (Kuru sizes in `pricePrecision` multiples) — dumping them would actively mislead intent alignment; curation is the point. Why this scales without per-protocol work in core: the audit plane needs no protocol knowledge (standard Transfer events + planned balance-diff backstop), and the observation plane's knowledge ships inside each protocol package, riding on ABIs the package already provides.

Deferred, recorded: the balance-diff audit backstop (account balance deltas via state-override eth_call, making asset-flow completeness vocabulary-independent and retiring the WETH9 special case), required canonical-receipt semantics (detecting a successful call that produced none of an expected standard transfer), and post-state Queries (running @Query against simulated post-state for position deltas).
